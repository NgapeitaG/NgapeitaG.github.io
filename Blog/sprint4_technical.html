Tell your non-tech friend a story about a time you:
  1. Got blocked on a simple problem
  2. Solved a problem in an elegant way.
  3. What did you learn?


1.

When I was learning the basics of Javascript with a website called Free Code Camp, I was stuck with a few issues which I had to solve. Free code camp is basically a website which gives you tutorials which you must follow and then you have to complete specific steps, so that you can learn how to use Javascript. One of the many challenges I had difficulty with whilst working through the FreeCodeCamp tutorials was using escape sequences in strings. I had a block when I didn't know where to put certain escape characters.

2.

To solve this problem, I followed the steps, carefully wrote out the code and went over it and ran the code. When the code didn't work constantly, I stopped, told myself that this is a simple problem which isn't difficult, I started over, re-typed my code and cracked the problem and found my solution. As long as you believe that you can solve a difficult problem and try again, you will get the answer. Breaking the problem down into smaller tasks and looking at each of those elements will help you look at the problem in smaller steps, so that you can successfully resolve your errors. Through out the process I felt frustrated sometimes but when I begin to feel like that, I take a breather, walk away from the computer to clear my mind and come back to the problem, only if I'm feeling stressed. Once I do resolve a coding error however, I feel like I'm achieving little by little.

3.

In this particular situation where I had to write the proper code using escape sequences, I learned that I can code and if I read between the lines, I'll figure out the solutions for various coding problems. It turns out variables are case sensitive, spacing is important so having no spacing in this case was required and how you type the text is just as important. In order for code to work correctly as a web developer/coder, you have to make sure that code whic you're writing out is formatted correctly. It's important that you type your text in correctly, otherwise it won't work, leaving you with errors which you have to fix along thr way.

Here is the problem which I had to write out: FirstLinenewlinetabbackslashSecondLinenewlineThirdLine
Here is the solution I resolved: var myStr = "FirstLine\n\t\\SecondLine\nThirdLine"

This may seem like a simple problem to resolve but what I was doing was over-thinking a simple solution and typing in the wrong solution. Don't over-think your code and look at how you're typing it.
